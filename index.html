JavaScript is a client-side scripting language. JavaScript runs in web browsers through the JavaScript engine. Different browsers use different JavaScript engines. Mozilla Firefox uses "Spider monkey", Google Chrome uses "V8", Microsoft edge uses "chakra". There is not a huge difference between this engine all the engines work very much the same. Whenever you place JavaScript code on a webpage and your browser loads that page, All browsers have a built-in interpreter that reads the code over javascript and runs it. JavaScript's source code is run in a web browser following some steps. Firstly The Source code is first received by the "parser" inside the JS engine. When the parser checks all the code and makes sure that there are no mistakes then it creates AST. After that, the code is converted to machine code. Lastly, the converted code is sent to the system for run.

There is not much difference between sessionStorage and localStorage, It works very similarly. The primary difference between them is data in local storage doesn't expire and session Storage's data expires. Whenever you visit a website, session storage data has been created. Which only works on that particular tab. Whenever you exit that tab. Session storage will also end. Local storage contains the data key-value pair format. If you turn off your browser or let's say turn off your desktop or laptop and after a long time opens your browser the local storage will remain in your browser. Local storage data cannot be deleted you have to delete it manually. This is the main difference between local storage and session storage. Local storage and session storage capacity up to 5MB. To view local storage and session storage data from Google Chrome, right-click on your mouse and open Inspect and go to the Applications tab. There you can see local storage and session storage.
